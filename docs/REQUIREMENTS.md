## 1. 项目概述

### 1.1 产品名称

**AI 热点监控**（AI Hotspot Monitoring）

### 1.2 一句话描述

一个多源聚合 + AI 驱动 + 赛博朋克风格的中文热点监控平台，**让用户在一处掌握全网正在发生什么、AI 已经替你想清楚它意味着什么**。

### 1.3 目标用户

- 内容创作者、自媒体从业者（需要快速捕获选题）
- 产品/运营/市场人员（需要洞察舆情）
- 投资人、研究员（需要追踪行业风向）
- 信息焦虑型重度用户（想用一个 App 替代 N 个 App）

### 1.4 核心价值主张

| 维度 | 传统聚合工具 | 本产品 |
|---|---|---|
| 数据 | 单平台 | **多平台聚合 + 自动跨源去重** |
| 排序 | 平台原生热度 | **AI 综合评分（热度 + 爆发速度 + 跨平台覆盖度）** |
| 洞察 | 仅标题 | **AI 摘要 + 分类标签 + 重要事件预警** |
| 体验 | 千篇一律 | **赛博朋克风格，沉浸式信息流** |

## 2. 功能需求

### 2.1 核心功能（必须有 / MVP）

#### 2.1.1 多源热点聚合

监控以下 **4 个平台**（第一阶段 X / Twitter 暂缓）：

| 平台 | 优先级 | 数据维度 |
|---|---|---|
| 微博热搜 | P0 | 排名、关键词、热度值、标签 |
| 知乎热榜 | P0 | 排名、问题、热度、回答数 |
| B 站热门视频 | P0 | 标题、播放数、点赞、UP主、分区 |
| GitHub Trending | P0 | 仓库名、星标、语言、描述 |
| ~~X / Twitter~~ | P2（暂缓） | 通过 twitterapi.io 接入，留接口 |

> **重要约束**：**多信息源并行**，避免单一来源失败导致整站静默。每条热点至少有 1 个源数据，去重后合并的同事件会保留所有源。

#### 2.1.2 AI 智能处理（OpenRouter）

对抓取到的原始数据执行 5 步 AI Pipeline：

| 步骤 | 输入 | 输出 | 模型 |
|---|---|---|---|
| 1. **分类** | 原始标题 + 上下文 | 类别（科技/社会/娱乐/财经/...） + 标签数组 | 默认 `google/gemini-2.5-flash` |
| 2. **去重** | 多源热点列表 | 合并同一事件的分组 | 同上（语义相似度） |
| 3. **评分** | 各源指标 + AI 判断 | 综合热度分（0-100） | 同上 |
| 4. **总结** | 标题 + 链接内容 | 200 字摘要 + 关键信息点 | 同上 |
| 5. **预警** | 用户订阅规则 + 评分 | 命中订阅 → 触发 Alert | （规则匹配，无 AI） |

#### 2.1.3 模型动态切换

- **设置页支持运行时切换** OpenRouter 上任意可用模型
- 默认提供 5-8 个推荐模型预设：
  - `google/gemini-2.5-flash` ⭐ 默认（速度 1.1s / 成本最低）
  - `google/gemini-3.5-flash`
  - `anthropic/claude-haiku-4.5`
  - `anthropic/claude-sonnet-4.5`
  - `openai/gpt-5.2`
  - `deepseek/deepseek-chat-v3.1`
  - `openai/gpt-5-nano`
- 用户也可自定义填入任意 OpenRouter 模型 ID

#### 2.1.4 多用户系统

- 注册 / 登录（邮箱密码 + GitHub OAuth）
- 个人账户独立的：订阅规则、预警记录、模型偏好
- 普通用户不可见管理后台

#### 2.1.5 订阅与预警

- 用户可创建多条订阅规则：
  - 关键词（如 "AI"、"OpenAI"）
  - 限定平台
  - 限定分类
  - 最低热度阈值（默认 70）
- 命中规则的热点自动生成 Alert
- 预警以**站内消息**方式呈现（第一阶段不做邮件/微信推送）

#### 2.1.6 抓取调度

- **默认每小时抓取一次**（cron `0 * * * *`）
- 支持管理员手动触发"立刻抓取"
- 单次抓取失败的源自动 fallback 到 **Firecrawl MCP**

### 2.2 非功能需求

| 维度 | 要求 |
|---|---|
| **视觉风格** | **赛博朋克**，不千篇一律，禁止使用 ShadCN 默认组件 |
| 性能 | 首屏 LCP < 2.5s，列表渲染 60fps |
| 可访问性 | 支持 `prefers-reduced-motion`（关闭动画） |
| 部署 | 本地优先（SQLite），后续可平滑切换到 PostgreSQL |
| 安全 | 用户密码 bcrypt 哈希；OpenRouter 系统 Key 仅服务端持有 |
| 国际化 | 仅中文（第一阶段） |

### 2.3 暂缓 / 不做

- ❌ 邮件 / 微信 / Telegram 推送（第一阶段仅站内）
- ❌ X / Twitter 数据源（第一阶段仅留接口）
- ❌ 移动端原生 App（仅做响应式 Web）
- ❌ 公共评论区
- ❌ 付费订阅 / 商业化
- ❌ 多语言

## 3. 用户故事

### Story 1：探索者小李

> 小李是科技自媒体作者，每天早晨打开 App，看到首页"今日 AI 类热点"被 AI 聚合成 8 条主题，每条都有 200 字摘要和跨平台来源。他点开一条 "OpenAI 发布 GPT-6"，看到它在微博、知乎、X、HN 都热议，省去他 30 分钟手动检索。

### Story 2：研究员小王

> 小王做行业研究，订阅了关键词 "量子计算"，阈值设置为 80。当任何平台出现相关热点且 AI 评分 ≥ 80，就会进入她的预警中心。她不需要每天主动刷。

### Story 3：高级用户小张

> 小张精通 AI，他想用 Claude Sonnet 4.5 做更深的洞察。他在设置里把默认模型切到 `anthropic/claude-sonnet-4.5`，看到摘要质量明显提升。

## 4. 约束与决策（来自用户问答）

| # | 决策项 | 用户选择 |
|---|---|---|
| 1 | 平台范围 | X、B 站、知乎、微博、GitHub（X 暂缓） |
| 2 | AI 能力 | 分类、去重、评分、总结、预警 |
| 3 | 视觉风格 | 赛博朋克 |
| 4 | OpenRouter Key | **系统级共享 Key**（管理员配置） |
| 5 | 用户系统 | **多用户**，需要登录注册 |
| 6 | 数据抓取频率 | 每小时一次 |
| 7 | 数据获取方式 | **多源**：爬虫 + Firecrawl MCP 兜底（X 用 twitterapi.io） |
| 8 | 模型切换 | **运行时动态切换** |
| 9 | 部署 | **本地优先**（SQLite） |
| 10 | 开发顺序 | **先网页版完成 → 再做 Agent Skills** |
| 11 | 技术对照 | **必须用 MCP 获取最新 API**，避免过时代码 |

## 5. 验收标准

### 5.1 阶段验收（Phase 6 完成 = 网页版上线）

- [ ] 4 个平台的热点能稳定抓取（手动触发 + 定时）
- [ ] AI Pipeline 五步全跑通，输出结构化数据
- [ ] 模型切换功能可用（至少能切换 3 个模型）
- [ ] 用户能注册、登录、退出
- [ ] 用户能创建订阅 + 接收预警
- [ ] 首页热点流呈现赛博朋克风格（与方案一致）
- [ ] 详情页能展示多源对比 + AI 总结
- [ ] 本地 `npm run dev` 一行命令启动通过
- [ ] README 完整，新人可在 10 分钟内启动项目

### 5.2 终极验收（Phase 7 完成 = Agent Skills 上线）

- [ ] 在 Cursor 中可通过 Skill 调用本系统能力
- [ ] Skill 文档完整，触发词清晰
